# 淘宝商品长图提取工具（.NET 8 / C# GUI）

版本：1.11.0  
日期：2026-07-10

## 环境要求

- Windows
- .NET 8 Desktop Runtime
- Google Chrome

程序不打包 .NET Desktop Runtime。发布目录会包含程序自身和 `Microsoft.Playwright` 依赖。

## 本机开发运行

启动 GUI：

```powershell
dotnet run
```

界面操作：

1. 检查或手动选择 Chrome 路径。
2. 点击“初始化”，程序会优先连接当前端口上的已有 Chrome；如果无法连接，才自动选择可用端口并启动 Chrome。
3. 在 Chrome 窗口中完成淘宝扫码登录。
4. 设置图片保存文件夹。
5. 在“自动获取”页签输入商品ID或商品链接，点击“开始获取长图”。
6. 也可以切换到“千牛源码辅助”页签：先转换千牛商品链接并复制到剪贴板，再粘贴商品页源码和店铺页源码，提取商品名、sellerId、shopId 后获取长图。
7. 双击“图片链接”列会复制图片链接网址；双击其他列会复制图片文件。
8. 复制图片文件后，可到目标文件夹中粘贴。
9. 如果获取失败，列表会保留已获取的商品名、sellerId、shopId；如果已生成店铺搜索页排查链接，该链接会显示在“图片链接”列，可双击复制。

## 发布给其他设备

```powershell
.\publish-win-x64.ps1
```

发布目录：

```text
publish-win-x64\淘宝商品长图提取工具
```

发布脚本会同时生成带版本号的压缩包：

```text
淘宝商品长图提取工具-v1.11.0-win-x64.zip
```

压缩包内会包含一层软件名称文件夹。解压后运行：

```text
淘宝商品长图提取工具\淘宝商品长图提取工具.exe
```

## 说明

- 程序会定位本机 Chrome 的常见安装路径。
- 可在界面中手动修改 Chrome 路径。
- 初始化时会优先复用当前端口上已打开的 Chrome。
- 当前端口不可连接时，如果端口被占用，会自动调整到下一个可用端口。
- 启动新 Chrome 后会确认调试端口已开放，避免界面误提示成功但后续连接失败。
- 不同端口会使用不同的独立 Chrome 用户资料目录，减少启动参数被旧实例吞掉的问题。
- 可在界面中设置图片保存文件夹。
- 界面支持“自动获取”和“千牛源码辅助”两套流程，可通过页签自由切换。
- 千牛源码辅助流程会把商品ID转换为千牛端商品链接并复制到剪贴板。
- 千牛源码辅助流程可从商品页源码提取商品名称，从店铺页源码提取 sellerId/shopId。
- 页面关键操作之间会加入固定范围内的随机等待，降低连续快速访问导致的误判风险。
- 商品标题会优先从 `span[class^="mainTitle"]` 的 `title` 属性提取；没有 `title` 时依次尝试 `value` 和文本。
- 任务列表默认按比例显示全部列，避免横向滚动条遮挡信息。
- 任务列表包含图片链接列，双击该列复制链接；双击其他列复制图片文件。
- 获取失败时，会保留已获取的商品名、sellerId、shopId；如果已生成店铺搜索页排查链接，会写入“图片链接”列，方便手动打开检查。
- 图片定位只使用商品名称拼接店铺搜索页，不使用商品ID作为搜索词。
- 如果店铺搜索页没有返回目标商品，程序会直接标记失败，不再使用商品详情页主图兜底。
- 因为连接的是本机已安装 Chrome，不需要下载 Playwright 自带浏览器。
- 不要把账号密码、Cookie、Session Token 或其他凭据提交到仓库。
